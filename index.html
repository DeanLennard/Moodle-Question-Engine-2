Moodle-Question-Engine-2 by timhunt

This is work-in-progress of my rewrite of the Moodle question engine (http://docs.moodle.org/en/Development:Question_Engine_2).

At the moment, there is not much to see. The interesting code is in http://github.com/timhunt/Moodle-Question-Engine-2/tree/new_qe2/question/engine/ and http://github.com/timhunt/Moodle-Question-Engine-2/tree/new_qe2/question/engine/interaction. The only way you can execute any of it is to checkout the new_qe2 branch, install Moodle as normal, and then run the unit tests in either question/engine or question/interaction. Code in http://github.com/timhunt/Moodle-Question-Engine-2/blob/new_qe2/question/interaction/deferredfeedback/simpletest/testwalkthrough.php would be a good place to start looking. This walks a single question through being attempted, using the API that an activity module using the question bank would use. Note that this is work in progress. Bits that are not finished: 1. Does not use the real question types yet. Uses fake code in question/engine/testquestiontype.php. 2. The way the renderers are currently implemented is not exemplary. They roughly work, but need refactoring. 3. The code in question/engine/lib.php is the heart of the new question engine, and is mostly solid.

Install

1. Checkout the new_qe2 branch. 2. Install Moodle as usual. 3. Run the unit tests in question/engine and question/interaction. 4. Look at the code in either of those folders.
